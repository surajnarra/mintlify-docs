## Overview

Emergent services use a fixed pool of outbound IP addresses for any traffic
that originates from the platform (for example, when pushing code to your
GitHub repository or calling an external API from a generated build). The
`/ip-addresses` page exposes the canonical list so operators can whitelist
them in firewalls, VPCs, or SaaS network rules.

34.61.212.41

These addresses are stable and will not change without notice. If Emergent
ever rotates them, the page will be updated before the cutover.

## When to whitelist

<CardGroup cols={2}>
  <Card title="GitHub Enterprise / self-hosted" icon="github">
    Add the range to your GitHub Enterprise allowlist so Emergent's GitHub app can push commits and read repository contents.
  </Card>
  <Card title="Private APIs" icon="server">
    If a generated build calls into a private service behind network ACLs, allow the outbound list to reach the service.
  </Card>
  <Card title="Webhook destinations" icon="bell">
    Accept inbound webhooks originated by Emergent by allowing the same outbound addresses.
  </Card>
  <Card title="SaaS tools with IP restrictions" icon="shield-check">
    Some tools (e.g. enterprise DBs) scope API access by IP. Whitelist this list to unblock MCP integrations.
  </Card>
</CardGroup>
