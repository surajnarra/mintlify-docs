## Overview

**Wingman** is Emergent's agent mode that trades the single-prompt
task flow for a more autonomous, longer-running build session. It runs inside
its own surface, reachable either from the App/Wingman switcher pill in the
top chrome or from the **Wingman** chip directly below the task input.

## How Wingman differs

<CardGroup cols={2}>
  <Card title="Higher agency" icon="wand-magic-sparkles">
    Wingman plans and executes longer sequences without waiting for user confirmation at each step. Suited to full-stack feature work.
  </Card>
  <Card title="Plan gated" icon="lock">
    Launching Wingman from either entry point opens the **Choose your plan** modal for non-paying accounts — Wingman is included with Standard and above.
  </Card>
  <Card title="Separate surface" icon="window-restore">
    Wingman occupies a distinct slot in the App builder chrome; it does not replace the standard App builder workflow.
  </Card>
</CardGroup>

## Entry points

Wingman can be launched from two places:

- **App/Wingman switcher** — the pill in the top-left chrome toggles between App builder and Wingman surfaces.
- **Wingman chip** — located directly below the main task input on the Home workspace.
