## Overview

The **Universal Key** is Emergent's provider-agnostic credential: one
`sk-emergent-…` token that can be used in any OpenAI-compatible SDK call and
will fan out to Anthropic, OpenAI, Google, or other supported providers based
on the model you request. It is managed entirely inside Account Settings and
ships with a separate pooled balance that auto-recharges from your main
account.

## Panels

<CardGroup cols={2}>
  <Card title="Key Details" icon="key">
    Shows the masked key (e.g. `sk-emergent-86••••`) with **Disable** and **Regenerate** actions. Disabling invalidates the key immediately; regenerating rotates it and invalidates the previous value.
  </Card>
  <Card title="LLM Providers" icon="diagram-project">
    Tab for enabling or disabling routing to individual providers. Useful for cost or compliance constraints when you want certain providers off the routing table.
  </Card>
  <Card title="API Key Balance" icon="wallet">
    Separate credit pool that API requests draw from. The default **Add Balance** action lets you top up without changing your subscription.
  </Card>
  <Card title="Auto Recharge" icon="arrows-rotate">
    Toggle that automatically refills the key balance by a configurable number of credits when it hits zero. The default refill amount is `5` credits.
  </Card>
</CardGroup>

## Safe rotation

<Steps>
  <Step title="Pre-stage a new key">
    Click **Regenerate** and copy the new value immediately — it is only fully visible once.
  </Step>
  <Step title="Update downstream consumers">
    Swap the key in any SDK config, environment variable, or secret manager that uses the Emergent endpoint.
  </Step>
  <Step title="Verify recharge settings">
    Confirm the API Key Balance and Auto Recharge thresholds match your expected workload before resuming traffic.
  </Step>
</Steps>
