## Overview

Emergent runs every build inside a tab. The **App builder** chrome at the top
of the page is the multi-tab shell that persists across navigation: the first
slot is always `Home`, and each remixed, previewed, or generated build opens in
its own closable tab to the right. A separate **Wingman** slot sits alongside
the App builder pill for the agentic higher-capability mode.

## Anatomy

<CardGroup cols={2}>
  <Card title="App / Wingman switcher" icon="arrows-left-right">
    Left-most pill toggles between the standard App builder and Wingman surfaces. Only one mode renders at a time.
  </Card>
  <Card title="Home tab" icon="house">
    Fixed tab that always points at `/home`. Cannot be closed; holds the primary task form and Showcase.
  </Card>
  <Card title="Build tabs" icon="window-restore">
    One tab per remix, preview, or active build. Each shows the project name and a close (✕) control.
  </Card>
  <Card title="New tab control" icon="plus">
    The `+` button opens an empty tab that inherits the active project's context.
  </Card>
</CardGroup>

## Lifecycle

<Steps>
  <Step title="Tab opens">
    Submitting a task or clicking **Preview** / **Remix this template** on a Showcase card creates a new tab and routes the browser to `/chat`.
  </Step>
  <Step title="Tab runs">
    The tab streams the live preview and agent log. Closing it does not cancel the underlying build; reopening reattaches.
  </Step>
  <Step title="Tab closes">
    Clicking the ✕ on a tab returns focus to the previously active tab (or Home if it was the only build open).
  </Step>
</Steps>
